Schemas - 

1. Customer (Name, DOB, Username, Password, Email, PhoneNumber)
2. Seller (SellerId, SellerName)
2. Product (ProductID, ProductName, Description, SellerId, Price, Category)
3. Order (OrderID, Username, ProductID, OrderDate)
4. Cart (Username, ProductID, Quantity)

User and Product are going to be defined in RDS, while Order and Cart are in DynamoDB.

For each microservice ...
Write the business logic (Node.js).
Create a Lambda function.
Define its data store (DynamoDB table or RDS schema).
Use API Gateway to create a REST endpoint (/users, /products, /cart, /orders).
Use Secrets Manager for any API keys (e.g., Stripe, SendGrid).

So the structure goes like:
service/
    model/
    routes/
    main.js

RDS -

-- Customers Table
CREATE TABLE customers (
    username VARCHAR(50) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    dob DATE,
    email VARCHAR(100) UNIQUE NOT NULL,
    phone_number VARCHAR(15),
    password_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Functions - 
1. Add a customer (POST /customer)
2. Get information about customer (GET /customer/{username})
3. Update customer details (PUT /customer/{username})
4. Delete customer details (DELETE /customer/{username})

-- Sellers Table  
CREATE TABLE sellers (
    seller_id SERIAL PRIMARY KEY,
    seller_name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Products Table
CREATE TABLE products (
    product_id SERIAL PRIMARY KEY,
    product_name VARCHAR(200) NOT NULL,
    description TEXT,
    seller_id INTEGER REFERENCES sellers(seller_id),
    price DECIMAL(10,2) NOT NULL,
    category VARCHAR(50),
    stock_quantity INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
